Nested files

The Product model stores product details. The OrderItem embedded document references a Product and includes quantity and total price. The Order model contains a list of OrderItem documents, representing the order's items. This setup efficiently organizes managing multiple products in single order.

Process automation

The Flask route automates order processing by tracking magazine and applying discounts. It checks product stock for order item, updates magazine, and calculates the total price. The order is then saved to the database.

  • - Iterates through form order items and fetches the corresponding product from the database.

  • - Checks if there is sufficient stock; if not, it flashes an error message and stops processing.

  • - Calculates the total price for each item, updates the product stock, and appends the item to the order.

  • -Handles the case where the product does not exist by flashing an error message.

  • -If the total order price exceeds 500, applies a 7% discount and flashes a success message.


Admin dashboard

The admin dashboard has several key features to help administrators analyze sales data. It ensures only admin users can access dashboard, blocking non-admin users. The dashboard selects the correct layout based on the user's role, showing a different view for admins and regular users. It finds the top three best-selling products by counting order items. It also calculates the number of orders placed on each date and the total revenue made on each date. This data is used to create charts with Chart.js, showing trends and insights. Additionally, it identifies the three least popular products based on how many were sold.

Security

User passwords are hashed using Werkzeug Security. During login, check_password_hash verifies the entered password against the stored hash. Plain-text passwords are never saved in the User collection.

Login managment

The application includes robust login management functionalities. Users can log in using their email and password through a secure login form. The login system checks the credentials against stored user data, ensuring secure access. Logged-in users can also log out, and there is support for different user roles, including administrators who have additional privileges.

Admin dashboard

Orders Count by Date: Displays the number of orders placed on each date, helping track sales trends over time.

Top Selling Products: Highlights the most popular products based on the quantity sold, providing insights into customer preferences.

Least Popular Products: Identifies products with the lowest sales, aiding in inventory management and marketing strategies.

Revenue by Date: Shows the total revenue generated on each date, providing a clear view of the business's financial performance.

How to add products and orders

Adding Products: Accessible via a dedicated form, users can input product details such as name, category, type, value, and price. This information is then saved to the database, making the new products available for orders.

Users can create new orders by filling out an order form, which includes customer information and order items. Each order item includes a product, quantity, and total price. The application ensures that sufficient stock is available for each product and calculates the total order price, applying a discount if applicable.

Products and orders searching

Users can search for products using a search query that matches the product name. The search results display relevant products, facilitating easy inventory management.

Users can search for orders by customer name or date. The application processes the search queries and returns matching orders, including the total quantity of items in each order.

PDF Downloading

For each order, users can download a detailed PDF summary. This feature generates a PDF document containing all relevant order details, such as customer information, order items, quantities, and prices. The PDF is formatted for clarity and includes a total order price and any applied discounts. This functionality helps in maintaining organized and accessible records of orders.

4. Paste in terminal -> source .venv/bin/activate
5. Paste in terminal -> pip3 install -r req.txt
6. Paste in terminal -> python3 app.py
